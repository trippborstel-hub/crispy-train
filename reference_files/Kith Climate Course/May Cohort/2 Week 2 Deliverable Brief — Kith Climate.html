Week 2 — Deliverable Brief

Data Quality Assessment &
Client Data Request

Your assignment this week: assess Northwood Capital's data environment and produce a structured data request for what's missing.

Scenario: Northwood Capital has hired your team. They've given you access to their data environment — files, reports, spreadsheets, memos. Like real client data, it's messy, incomplete, and sometimes contradictory.

Last week's pitch deck was the easy part. This week you build the analytical foundation.

Your Five Deliverables

1
Identify what data exists
Inventory the environment. What did Northwood give you? Catalog every file, dataset, and document. Note what type each is (structured, unstructured, code) and where it lives.
2
Assess its quality
Completeness, consistency, reliability. Where do you trust it? Rate each data source and flag anything that looks suspect. A consultant who trusts bad data produces bad work.
3
Determine fixes for data errors
Contradictions, unit mismatches, outliers. What's the correction? Document each error you find, explain why it's wrong, and propose a specific fix or reconciliation.
4
Identify missing data
Gaps that block analysis. What do you need that isn't there? Think about what a complete climate/ESG assessment requires and map the gaps between that and what you have.
5
Create your data ask
A structured request back to the client for what's missing. This is a professional document — prioritized, specific, and written so a busy client team knows exactly what to send you and why.

How to approach this

  • Open your Northwood Capital environment in Claude Code and start by asking it to inventory what's there
  • Let Claude Code read the data — structured, unstructured, and code files all at once. No connectors. No ETL.
  • Direct Claude Code to cross-reference sources against each other to find contradictions and inconsistencies
  • Use the Inputs → Outputs → Inputs pattern: your inventory becomes the input for your quality assessment, which becomes the input for your data ask
  • Think like a consultant — your data ask should be something you'd actually send to a client

Suggested Output Formats

You choose the format. The deliverable should be something you'd present to a client or a project lead. Here are some natural fits:

Data Inventory
Spreadsheet (.xlsx) or structured markdown table
Quality Assessment
Scorecard with ratings per data source
Error Log
Table of findings with proposed corrections
Data Ask
Professional memo or structured request document
