Getting Started with Claude Code

Follow these steps before your first Monday session. Most take under 5 minutes. If you hit a snag, Step 4 has everything you need to troubleshoot.

1

Create a Claude Account

You need a Claude account to use Claude Code. Head to claude.ai and sign up. Once registered, choose a plan:

Not sure which? Start with Pro. You can always upgrade later if you hit usage limits.

2

Set Your Privacy Preference

Before installing Claude Code, take 30 seconds to set your training preference. By default, Anthropic may use your conversations to train future versions of Claude. For cohort work — which often touches company context, NDA-bound information, or client data — we recommend turning this off.

How to do it:

  • Open claude.ai and sign in.
  • Click your profile icon at the bottom-left → Settings.
  • Find Privacy in the left sidebar.
  • Toggle OFF "Allow your chats to be used to train Claude Code".

Want the full picture? A separate Claude Privacy & Data Security cheat sheet ships with the cohort materials and walks through retention periods, telemetry controls, and other privacy settings worth knowing about.

3

Download the Claude Code Desktop App

Download the Claude Code desktop application from Anthropic's site:

claude.ai/download

Install and open it. Having this done before Monday's session is excellent, but by no means required — we will walk through connecting everything together in our first session.

New as of May 2026: the desktop app bundles Claude Code. Once installed and signed in, the Code tab at the top of the app gives you full Claude Code access — no terminal install needed. Step 4 below is now optional.

4

Install Claude Code via Your Terminal * optional

* Optional for cohort use. The desktop app's Code tab (from Step 3) already gives you full Claude Code functionality. This terminal install is only needed if you want to run claude directly from your terminal — useful for power users and developers, but not required to complete the cohort.

If you'd like to set up the CLI anyway, open a terminal on your computer and run the install command for your operating system.

How to open your terminal:

  • Windows: Search for PowerShell in the Start menu and open it. Alternatively, press Win + X and select PowerShell.
  • macOS: Press Cmd + Space to open Spotlight, type Terminal, and hit Enter.
  • Linux: Press Ctrl + Alt + T or find Terminal in your applications menu.

Run the install command:

 macOS / Linux
curl -fsSL https://claude.ai/install.sh | bash
⊞ Windows (PowerShell)
irm https://claude.ai/install.ps1 | iex

Once the install finishes, type claude and press Enter. If you see a welcome prompt, you're all set.

5

Troubleshooting

If the install command returns an error, don't worry. Your first step is to ask Claude itself for help — open a chat at claude.ai, paste the error from your terminal, and it will walk you through the fix. Copy and paste the prompt below to get started:

Copy & paste this into claude.ai
I am currently working to download Claude Code on my computer. I have a Pro subscription and the desktop app downloaded. I ran the Claude Code install command in my terminal, but faced an error. Please see the error output below and the screenshot attached.

[Paste your terminal error here]

Tip: You can also attach a screenshot of the error. On Windows, press Win + Shift + S to capture a screenshot. On macOS, press Cmd + Shift + 4.

Common downloads you may need

On Windows, the most common missing dependency is Git. The direct link for convenience:

Git for Windows
git-scm.com/downloads

Download and install using the link above (accept all defaults during installation), then try the install command from Step 3 again.

Already up and running? If Claude Code launches but something feels off later in the program, try running /doctor inside Claude Code. It is a built-in diagnostic that checks your install and surfaces any issues without needing the terminal.

6

Create Your Project Folder

Once Claude Code is working, you need to create a dedicated folder where all your work will live. Open your terminal and run the command for your operating system:

 macOS / Linux
mkdir ~/claude-projects && cd ~/claude-projects
⊞ Windows (PowerShell)
mkdir "$env:USERPROFILE\claude-projects"; cd "$env:USERPROFILE\claude-projects"

This creates a folder called claude-projects in your home directory. This is where you'll store all the files and projects you build during the program.

Where is this folder?

  • Windows: C:\Users\YourName\claude-projects
  • macOS: /Users/YourName/claude-projects

Why this specific location? By placing your project folder at your user home — not inside Documents, Desktop, or a cloud-synced folder — Claude Code gets a clean, contained workspace. It can only read what's in this folder. Your personal documents, work files, photos, and corporate data stay outside its reach.

Don't move this folder into OneDrive, iCloud Drive, Dropbox, or Google Drive. The commands above placed it in the right spot — at your user home, next to your sync folders, not inside them. Leave it there.

Two reasons: cloud sync chokes on the thousands of small files Claude generates (your machine will slow noticeably), and a synced folder can surface files to Anthropic that you didn't intend to share.

7

Connect Your Folder in Claude Code Desktop

Open the Claude Code desktop app. When it asks you to select a working directory, navigate to the claude-projects folder you just created.

From here, you can download any files provided by our team and ask Claude Code to add them into this folder. This will be your workspace for the first week of the program.

You're ready. This will get you set up for the first week — you can now start working on your deliverables.

FAQ

Answers to the questions that have come up most often. Read these before Monday and you will skip a lot of friction.

01I see three tabs at the top of the app — Chat, Cowork, and Code. Which one do I use?

Code. The Code tab is Claude Code — what we use throughout the program.

Chat is the regular claude.ai conversation interface. Cowork is a more pre-packaged experience with bundled skills. Neither of those is what you want for this cohort. We are building from scratch, not using training wheels.

02The send arrow at the bottom of the chat is greyed out. Did something fail?

No. The send arrow stays grey until you select a working folder. Pick your claude-projects folder from the folder toggle at the bottom of the window, and the arrow will brighten.

This caught a lot of people in the previous cohort. If your install otherwise looks normal but the arrow is grey, the folder is the fix.

03Claude is asking my computer for permission to access folders outside claude-projects. What do I do?

Decline. Claude Code only needs access to your claude-projects folder for this program. If a popup asks about Documents, Downloads, Desktop, or anything else, click no.

This keeps Claude's scope clean and your personal files private. You can always grant additional access later if a specific task needs it.

04I see a message saying "you've hit your usage limit." Did I break something?

No — this is the normal Pro plan behaviour. Tokens reset every 4 hours. Large data dumps and "read everything in this folder" prompts burn through tokens quickly.

If you hit the limit, wait for the reset rather than upgrading on the spot. We will cover the /compact command and other token-saving techniques in Week 2 — those make a big difference once you know them.

05Can I use my iPad or phone for the program?

No — Claude Code is a desktop application. You will need a laptop or desktop running Windows 10 or later, macOS 13 or later, or Linux. 8 GB of RAM minimum (most modern machines have plenty).

iPads cannot run the terminal commands required to install Claude Code.

06What format should my deliverables be in?

HTML, by default. When asking Claude to produce a slide deck, brochure, dashboard, or report, ask for it in HTML first. Claude renders HTML quickly and beautifully. You can always convert HTML to PDF afterwards (just ask Claude to do it).

Avoid asking for PowerPoint or Word directly. Claude will technically build them, but the workflow involves writing format-specific code, downloading conversion tools, and walking through dozens of extra permission prompts. HTML first saves you a lot of time.

07What is CLAUDE.md and do I need to create one?

CLAUDE.md is a plain-text Markdown file Claude reads automatically at the start of every session. Think of it as standing instructions for how Claude should work inside that folder — your rules, your standards, anything you find yourself repeating.

You do not need to create one manually. Claude often creates one for you the first time you start a session in a new folder. When it does, open it and skim what is inside. Edit it freely — over the six weeks this file becomes your most valuable piece of IP.

The filename has to be exactly CLAUDE.md. Not claude2.md, not a Word document — it has to be the Markdown file with that exact name for Claude to auto-read it.

08Can I interrupt Claude in the middle of a long task?

Yes. Press Esc or just type a new message — Claude will pause, respond to you, and then continue. It does not lose the work it was doing.

If you want to halt fast (you spotted something wrong), type WAIT in all caps. Claude stops immediately and asks what is up.

09I installed a new tool but Claude Code does not seem to see it. Help?

Fully quit Claude Code and reopen it. The X button often just hides the window — Claude keeps running in the background.

Windows: open Task Manager, find Claude Code, right-click → End Task. Then relaunch.

macOS: right-click the Claude icon in the Dock and select Quit, or press Cmd + Q while focused on the app. Then relaunch.

10What is the difference between the + button and the folder picker?

They do different things and the UI is easy to confuse.

The folder toggle at the bottom of the window sets your working folder — the place Claude reads from and writes to for the whole session. This is the one you want for the cohort.

The + button next to the chat input attaches one-off files to a single message — like dragging an image into an email. Use this only when you have a specific file you want Claude to look at right now.

11Something is broken and I do not know where to start. What now?

Two things, in order:

1. Run /doctor inside Claude Code. It is a built-in diagnostic that checks your install, your tools, and your config, then tells you what is wrong in plain English.

2. If you are still stuck, paste the error into claude.ai chat with the prompt template from Step 4 above. Claude will walk you through the fix.

And if both of those leave you stuck — post in the cohort Discord. You are not the only one.
