Week 6
Why AI Can Provide Assurance
The concepts behind the evidence chain you’re building
Carbon Accounting Is Estimation
This has always been true. It matters for what comes next.
Assurance does not attest that a number is exactly correct. It attests that the methodology is sound, the estimates are reasonable, and the assumptions are documented.
The question about AI is not whether it introduces estimation into a precise process. Estimation was always the process.
The benchmark is reasonableness and documentation — not numerical perfection.
Two Kinds of Error
Most AI-risk conversations treat all error as equivalent. The distinction matters.
Random Error
Non-directional. Does not reproduce. Washes out.
Exactly what sampling and spot-checks catch.
Systematic Bias
Directional. Recurs for the same reason. Accumulates.
Replication does not remove it. This is the real threat.
The rest of this deck is about where systematic bias can live — and where it can’t.
Three Places Bias Can Enter
Source Data
The client’s documents are wrong. Same exposure as manual. Not an AI question.
Orchestration
A shared prompt tilts every agent the same way. Real — but correctable. You can read the prompt, find the bias, fix it.
The Weights
An encoded prior in the model itself. The only locus of uncorrectable systematic bias.
Two are either shared with manual processes or fixable. The third is where we need to be precise.
Most of the Work Is Not Vulnerable
Extraction
Perception, not belief. No directional way to consistently misread a kWh figure.
Classification
Settled convention. Electricity → Scope 2 is not a judgment call.
Calculation
The model doesn’t do arithmetic. A deterministic engine does.
These tasks fail visibly, on the first test. Systematic bias doesn’t lurk here.
The Honest Carve-Out: Estimation
When a system fills a data gap, the model’s prior does the work. Estimates regress toward the mean of the training distribution. For an atypical client, that’s a directional skew.
This is real bias. But it is also the one sub-task the system tags as an estimate, documents the methodology for, and surfaces for human judgment.
The claim is not “no bias.” It’s that the only uncorrectable bias lives where the system concentrates scrutiny.
“Independence” Is the Wrong Word
“Agent B checks Agent A” is not independent if both share the same priors.
Error-Finding Framing
Ask QA to find errors, not confirm correctness. Removes the confirmation reflex.
External Referents
Check against something outside the model: stored source docs, versioned factor APIs, deterministic calcs.
The Human Layer
Ambiguous cases flagged for human review. A named preparer owns the assertion.
External referents don’t share the model’s priors. That’s where real independence comes from.
Provenance Is a Binding, Not an Assertion
Not the model’s claim about where a value came from. A binding to a stored, hashed source document that any process can verify.
A fabricated value has nothing to bind to — it fails the check because the referent sits outside the model. Source-binding turns hallucination from a philosophical worry into a failed check.
Fabrication is the failure mode unique to generative systems. It is also the one most directly defeated by architecture.
The Claim Is Testable
“No systematic bias” is not something you assert. It is something you demonstrate.
Back-test against ground truth. Show residuals are mean-centred — random scatter, not directional drift. Compare estimates against subsequently metered actuals. Check for skew.
A bar worth holding the manual baseline to. When did a spreadsheet process last demonstrate its error was centred?
Your Work This Week
QA refs
Trace values to source documents. That’s the external-referent layer.
Process refs
Document the methodology. Makes orchestration-level bias discoverable.
Estimates
Tag them. Document the inputs. Surface where bias can live.
The reference library is not just process documentation. It is an assurance architecture.
The Core Insight
A well-architected AI system doesn’t eliminate error.
It makes every figure testable.
Traceable to source. Independently recomputable.
Checkable for the one thing that actually threatens an inventory.
