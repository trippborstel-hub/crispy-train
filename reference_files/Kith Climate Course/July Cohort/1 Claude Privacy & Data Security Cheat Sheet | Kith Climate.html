01

Consumer vs. Commercial: Two Different Worlds

Your plan type determines your privacy protections. This is the single most important distinction.

Consumer  Free / Pro / Max

  • Training on your data: Opt-in toggle — you choose
  • Retention (training on): Up to 5 years (de-identified)
  • Retention (training off): 30 days
  • Safety-flagged content: Retained 2 years regardless
  • Trust & safety scores: Retained 7 years
  • Staff chat access: Limited — policy review or explicit consent
  • Data sold: Never

Commercial  Team / Enterprise / API / Gov / Education

  • Training on your data: Never (unless opted into Dev Partner Program)
  • API retention: 7 days
  • Team/Enterprise retention: 30 days
  • Zero Data Retention (ZDR): Available for Enterprise
  • Staff chat access: Same limited policy
  • Data sold: Never
Key Takeaway

A "Pro" plan sounds professional, but it is a consumer account with consumer-level privacy. Only Team, Enterprise, or API access provides business-grade data handling.

What "deleted" means

  • Deleted conversations are removed from your chat history immediately but remain on back-end systems for up to 30 days
  • Deleted chats are never used for training under any circumstance
  • If you turn training off later, Anthropic cannot "unlearn" data already incorporated into models
  • Training ON only applies to new/resumed chats after enabling — not retroactively to older inactive chats
03

Claude Code: Larger Data Surface Area

Every file Claude Code reads is sent to Anthropic's servers. This creates a significantly larger data footprint than the chat interface.

Local storage

  • Session transcripts stored in plaintext (JSONL) at ~/.claude/projects/ for 30 days by default
  • Anyone with machine access can read these files
  • Adjust retention with the cleanupPeriodDays setting

Server-side retention

  • Follows your account type: 30 days (consumer), 7 days (API), or ZDR (enterprise)
  • Data encrypted in transit via TLS
  • Data is not encrypted at rest on Anthropic's servers

Telemetry controls

Environment Variable What It Disables
DISABLE_TELEMETRY=1 Statsig operational metrics
DISABLE_ERROR_REPORTING=1 Sentry error logging
DISABLE_FEEDBACK_COMMAND=1 The /feedback command
CLAUDE_CODE_DISABLE_FEEDBACK_SURVEY=1 Session quality surveys
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1 All of the above at once
Watch Out — /feedback

Using /feedback sends a copy of your full conversation history including code to Anthropic, retained for 5 years. Avoid using it on sessions with sensitive content.

04

Why Secrets Are Riskier in Claude Code

Both interfaces store data server-side under the same retention rules. The difference is the attack surface.

Claude Code — Higher Exposure

  • Automatic file reading captures .env files, config, and hardcoded keys without you realizing
  • Plaintext transcripts at ~/.claude/projects/ are readable by anyone on the machine
  • Memory propagation can extract secrets from transcripts and inject them into future sessions
  • /feedback sends full session (incl. secrets) — retained 5 years

Claude.ai Chat — Lower Exposure

  • Manual control — you decide exactly what enters the conversation
  • No filesystem access — uploads are explicit and intentional
  • No local transcripts — no plaintext session data on your machine
  • Same server-side rules still apply — avoid sharing secrets regardless
Best Practice

Never share production secrets, API keys, or data under NDA in either interface. Use placeholder values or environment variable references instead.

05

The API Is the Strongest Privacy Option

API access operates under Commercial Terms regardless of whether you pay personally or through an organization.

  • 7-day retention vs. 30 days or 5 years on consumer plans
  • Never used for training — no consumer data policy applies
  • Claude Code inherits these protections when authenticated with an API key instead of a consumer login
  • This is the simplest privacy upgrade for anyone doing sensitive work without an enterprise contract

07

Claude Projects & Memory

Additional data surfaces beyond individual conversations:

  • Projects: Custom instructions and uploaded files are stored persistently for as long as the project exists, and are sent with every conversation in that project
  • Memory (Claude.ai): Claude can store facts across conversations. This data persists until you manually clear it
  • Claude Code memory: Local CLAUDE.md and memory files are read and sent to the server with each session
Note

Project files and memory contents are subject to the same retention rules as conversations for your account type. Review what's stored in your projects periodically.

08

Compliance & Certifications

  • SOC 2 Type II: Anthropic holds SOC 2 Type II certification
  • HIPAA: Business Associate Agreements (BAAs) available for Enterprise customers
  • Data residency: Not currently offered — data is processed and stored in the US
  • GDPR: Standard contractual clauses and DPA available for applicable customers
Quick Reference

Privacy Checklist

  1. Check your training toggle. Claude.ai → Settings → Privacy → "Improve Claude for everyone." Turn it OFF if you don't want data used for training.
  2. Delete sensitive conversations when done. Deleted chats are purged within 30 days and never used for training.
  3. Use API access for sensitive work. Even a personal API key gives you 7-day retention and no-training guarantees.
  4. Never put production secrets in Claude Code. Use placeholders in config files. Clear local transcripts at ~/.claude/projects/ periodically.
  5. Push for commercial plans for business use. Pro is still consumer. Only Team, Enterprise, or API provides business-grade protections.
  6. Disable Claude Code telemetry if privacy matters. Set CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1.
  7. Audit for shadow AI. Check if team members are using personal Free/Pro accounts for work — their data may be subject to training and 5-year retention.
  8. Review Projects & Memory. Periodically audit what's stored in Claude Projects and memory features.
